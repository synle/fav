# fav — Architecture

## High-Level Overview

`fav` is Sy's personal favorites / bookmarks launcher, deployed as a static
single-page site at https://synle.github.io/fav/. There is no backend, no
build framework, and no runtime dependencies — just vanilla HTML and JavaScript
served as static assets from GitHub Pages.

Runtime model:

- The repo ships an `index.html` shell and an `index.js` "schema" file.
- The shell `index.html` is regenerated by `build.sh` from an upstream template
  hosted at `synle.github.io/nav-generator/index.template.html`. The repo
  itself does not author the shell directly.
- At page load, `index.html` pulls in
  `https://synle.github.io/nav-generator/index.js?hasCustomNavBeforeLoad=1`,
  which renders the navigation UI and dispatches a `NavBeforeLoad` event.
- `index.js` listens for `NavBeforeLoad`, assembles a multi-section schema
  string (built-in favorites, Android list, URL Porter bookmarks, RVX configs,
  IP / hosts config), and hands it back to the nav generator's `renderSchema`
  callback for rendering.
- A service worker (`sw-nav.js`) is registered from the same nav-generator
  origin for offline support and hourly update checks; new versions trigger
  an auto-reload via `controllerchange`.
- URL Porter integration is two-way: `url-porter.json` lives in the repo
  (Chrome extension config — short alias → URL mappings), and a browser
  extension can post a `urlPorterBookmarks` DOM event into the page at load
  time to inject additional bookmark entries.

The repo is, in effect, a configuration payload — the heavy lifting (rendering,
service worker, search-as-launcher) lives in `synle/nav-generator`.

## Key Directories

- `/` (root) — All source. Flat layout; no subdirectories for code.
- `.github/workflows/` — CI: build-on-push and PR-artifact cleanup. Both are
  thin wrappers around reusable workflows in `synle/workflows`.

## Important Files

- `index.html` — Page shell. Overwritten by `build.sh` from the nav-generator
  template. Registers the service worker and loads the nav-generator script.
  Do not hand-edit; regenerate via `npm run build`.
- `index.js` — The interesting file. Defines the `SITE_SCHEMA` (sections,
  tabs, link rows in the nav-generator's mini-DSL), the `getStrongPassword`
  utility (dispatched via `AppCopyTextToClipboard`), and the `NavBeforeLoad`
  handler that composes the final schema (favorites + Android list + URL
  Porter bookmarks + URL-Porter / RVX / Nav-Generator / IP tabbed "Tools &
  Configs" section + extension-injected bookmarks).
- `url-porter.json` — URL Porter Chrome extension config: `{ homepage,
configs: [[alias, url], ...], bookmarkRules?, ... }`. Loaded at runtime
  via `fetch("https://synle.github.io/fav/url-porter.json")`.
- `url-porter.clean-config.js` — Node script invoked by `build.sh`. Reads
  `url-porter.json`, strips comments, normalizes each entry into a
  `[from, to]` pair (lowercase, trimmed, `http://` prefix added when scheme
  is missing, trailing slashes removed), deduplicates, and rewrites the file
  in place. Preserves optional keys (`bookmarkRules`, `historyAliasLimit`,
  `statsMaxResults`, etc.) if present.
- `build.sh` — Three-step build: download `index.html` template, download
  `dev.sh`, then run `node url-porter.clean-config.js`. Idempotent.
- `dev.sh` — One-liner: `curl` a generic dev-server bash script from
  `synle/workflows` and pipe it through `bash -s --` with watch globs
  (`*.json *.scss *.jsx *.js`) and the start command (`npm run start`).
- `package.json` — `private: true`, version `1.2.0`. Only dev dep is
  Prettier. Scripts: `clean`, `build`, `format`, `dev`, `start` (the last
  shells out to `npx http-server --cors -a 127.0.0.1 .`).
- `README.md`, `DEV.md`, `CLAUDE.md` — Reader-facing docs. `CLAUDE.md`
  pins the GitHub raw-URL format (`?raw=1` blob URLs) and the squash-merge
  policy.

## Build & Release Flow

Local:

1. `npm ci` (or `npm install`) — installs Prettier only.
2. `npm run build` runs `build.sh`, which:
   - `curl`s the nav-generator template into `index.html`,
   - `curl`s `dev.sh` into the repo,
   - runs `node url-porter.clean-config.js` to normalize `url-porter.json`
     in place.
3. `npm run dev` starts a watcher + static server via the `synle/workflows`
   `dev.sh` script. `npm start` runs a plain `http-server` on `127.0.0.1`.
4. `npm run format` runs Prettier with `--print-width 140` and a local cache.

CI (`.github/workflows/build-main.yml`):

- Triggers on push to `main`/`master` and on PRs targeting them.
- Delegates entirely to the reusable workflow
  `synle/workflows/.github/workflows/build-and-commit-sh.yml@main`, which
  runs `build.sh`, commits any drift back to the branch, and publishes to
  GitHub Pages (`contents: write`, `pages: write`, `id-token: write`).
- Concurrency group keyed on `workflow + ref` with `cancel-in-progress`
  so superseded runs are dropped.

PR hygiene (`.github/workflows/cleanup-pr-artifacts.yml`):

- Triggers on PR `closed` (merged or not).
- Delegates to `synle/workflows/.github/workflows/cleanup-pr-artifacts.yml@main`
  with `delete-runs: false` — i.e. expire artifacts, keep run history.

Release: there is no version-tag release pipeline. "Release" here means a
merged PR to `main`, which the build workflow auto-deploys to GitHub Pages.
The `version` field in `package.json` is informational only.
